# Reglas de construcción (definidas)

## Contrato operativo
1. PRD primero.
2. Cambios pequeños.
3. Validación técnica mínima.
4. Evidencia de aceptación.
5. Trazabilidad en commit/PR.

## Gate de calidad mínimo
- Build exitoso
- Caso feliz validado
- Caso borde validado
- Sin secretos
- Referencia PRD en PR

## Herramientas IA
- Codex: implementación compleja (con AGENTS.md)
- Antigravity: apoyo táctico (con GEMINI.md)

## Resultado esperado por iteración
- Algo funcional, medible y demostrable
- Sin deuda oculta crítica
- Con rollback simple
