# Context Changelog

Registro cronológico de cambios funcionales y decisiones de implementación.

## Formato obligatorio por entrada
- Fecha: YYYY-MM-DD HH:mm (TZ)
- Autor: nombre/agent
- Tipo: `feature | fix | decision | scope | risk | rollback`
- PR/Commit: referencia
- Sección PRD impactada: sección exacta en `context/PRD.md`
- Resumen (1-3 líneas)
- Impacto: qué cambia para equipo/usuario
- Riesgo/mitigación (si aplica)
